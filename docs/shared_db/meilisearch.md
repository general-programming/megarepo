# Meilisearch (sea1)

Meilisearch is deployed in the `shared-db` namespace on sea1 and exposed on Tailscale as `svc-meilisearch-sea1.tail285d8f.ts.net` on port `7700`.

The master key is generated by Terraform (`terraform/auth/meilisearch.tf`) and stored in Vault at `secret/app/meilisearch/sea1`. It is synced into the cluster as the `meilisearch-master-key` secret in `shared-db`.

## Prerequisites

- Connected to Tailscale
- `vault` CLI authenticated, or access to the Vault UI

## Fetching the master key

```sh
vault kv get -field=MEILI_MASTER_KEY secret/app/meilisearch/sea1
```

Export it for the commands below:

```sh
export MEILI_MASTER_KEY=$(vault kv get -field=MEILI_MASTER_KEY secret/app/meilisearch/sea1)
export MEILI=http://svc-meilisearch-sea1.tail285d8f.ts.net:7700
```

## Creating a search-only API key

To create a scoped key that can only search a specific index:

```sh
curl -X POST "$MEILI/keys" \
  -H "Authorization: Bearer $MEILI_MASTER_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "description": "lots-search-key",
    "actions": ["search"],
    "indexes": ["lots"],
    "expiresAt": null
  }'
```

The response contains the `key` field — that is the value to hand to clients.

## Listing existing keys

```sh
curl -s "$MEILI/keys" \
  -H "Authorization: Bearer $MEILI_MASTER_KEY" \
  | jq '.results[] | {description, key, actions, indexes}'
```

## Storing a key in Vault

After creating a key, store it in Vault so other services can consume it via `VaultStaticSecret`:

```sh
vault kv put secret/app/meilisearch/keys/lots-search \
  MEILI_SEARCH_KEY=<key value from above>
```

## Health check

```sh
curl -s "$MEILI/health"

```
